# 시작하기

이 가이드는 새 환경에서 전체 화면 TUI를 사용해 첫 로컬 Markdown 동기화까지 가는 가장 빠른 흐름을 설명합니다.

## 1. `syncdown` 설치

`syncdown`이 아직 `PATH`에 없다면 먼저 사전 빌드된 바이너리를 설치하세요.

`syncdown`은 다음 릴리스 바이너리를 제공합니다.

```text
darwin-arm64
darwin-x64
linux-x64
windows-x64
```

macOS와 Linux:

```sh
curl -fsSL https://raw.githubusercontent.com/hjinco/syncdown/main/scripts/install.sh | sh
```

Windows PowerShell:

```powershell
irm https://raw.githubusercontent.com/hjinco/syncdown/main/scripts/install.ps1 | iex
```

수동 다운로드가 필요하다면 [GitHub Releases page](https://github.com/hjinco/syncdown/releases)에서도 받을 수 있습니다.

## 2. TUI 실행

인터랙티브 터미널을 열고 다음 명령을 실행합니다.

```sh
syncdown
```

그러면 OpenTUI 앱이 열립니다. 여기에는 다음이 포함됩니다.

- 출력 경로와 커넥터 인증을 설정하는 화면
- 진단 정보와 시스템 경로
- 커넥터 실행과 로그 확인을 위한 동기화 대시보드

TTY 없이 `syncdown`을 실행하면 UI 대신 도움말이 출력됩니다.

## 3. 출력 디렉터리 선택

TUI에서 **Output**을 열고 프리셋을 선택하거나 사용자 지정 경로를 입력합니다.

`syncdown`은 저장 전에 경로를 정규화하고, 렌더링된 Markdown을 그 디렉터리 아래에 기록합니다.

이 출력 트리 아래의 Markdown 파일과 커넥터 폴더는 `syncdown`이 관리하는 출력물로 보는 편이 좋습니다. 생성된 `.md` 파일이나 폴더를 직접 수정하거나, 이름을 바꾸거나, 이동하거나, 구조를 다시 정리하는 것은 권장하지 않습니다. 이후 동기화나 전체 재동기화 과정에서 다시 생성되거나 덮어써지거나 삭제될 수 있습니다.

## 4. 소스 연결

### Notion

**Connectors → Notion**을 열고 다음 흐름 중 하나를 선택합니다.

- **Connect with Token**: `syncdown`이 Notion integrations 페이지를 열고 integration token 입력을 요청합니다.
- **Connect with OAuth**: `syncdown`이 Notion public integration client ID와 client secret을 요청하고, 브라우저를 연 뒤 로컬 콜백을 기다립니다.

### Gmail

**Connectors → Gmail**을 열고 **Connect Google account**를 선택합니다.

TUI는 Google Desktop app client ID와 client secret을 요청하고, 브라우저를 열어 로컬에서 OAuth 콜백을 완료합니다.

### Google Calendar

**Connectors → Google Calendar**를 엽니다.

공유 Google 계정이 아직 연결되지 않았다면 **Connect Google account**를 사용하고, 그다음 **Select calendars**를 열어 동기화할 캘린더를 선택합니다.

## 5. 첫 동기화 실행

TUI 홈 화면에서 **Sync**를 열고 다음 중 하나를 선택합니다.

- **Run all**
- **Run Notion**
- **Run Gmail**
- **Run Google Calendar**

최근 커넥터 로그와 실행 상태는 같은 대시보드에서 계속 확인할 수 있습니다.

대신 헤드리스 CLI 루프를 쓰고 싶다면 다음을 실행합니다.

```sh
syncdown run --watch
syncdown run --watch --interval 5m
```

`--interval`을 생략하면 기본값은 `1h`입니다.

## 6. 결과 확인

출력 디렉터리에서 커넥터별로 정리된 Markdown 파일을 확인합니다.

CLI에서 현재 앱 상태를 확인할 수도 있습니다.

```sh
syncdown status
syncdown connectors
syncdown doctor
```
