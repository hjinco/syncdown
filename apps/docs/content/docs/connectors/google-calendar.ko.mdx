# Google Calendar

## 인증 모델

Google Calendar 커넥터는 공유 Google OAuth 계정을 사용하며 Calendar readonly scope가 필요합니다.

```text
https://www.googleapis.com/auth/calendar.readonly
```

Gmail도 활성화되어 있다면 `syncdown`은 같은 Google 계정에서 Gmail과 Calendar scope의 합집합을 요청합니다.

## 여기서 OAuth가 의미하는 것

OAuth를 사용하면 `syncdown`과 Google 계정 비밀번호를 공유하는 대신 브라우저에서 캘린더 접근을 승인할 수 있습니다.

Google Calendar에서는 다음을 의미합니다.

- Google **Desktop app** OAuth client를 만들거나 기존 것을 재사용합니다.
- client ID와 client secret을 `syncdown`에 입력합니다.
- `syncdown`에 필요한 Calendar scope에 대해 Google이 브라우저 동의 화면을 표시합니다.
- 이후 동기화 때마다 다시 연결하지 않아도 되도록 `syncdown`이 연결을 완료합니다.

Google OAuth 계정은 Gmail과 Google Calendar 커넥터가 공유합니다. 필요한 scope만 모두 갖추고 있다면 하나의 Google 계정으로 두 커넥터를 모두 만족할 수 있습니다.

## TUI 설정

**Connectors → Google Calendar**를 엽니다.

공유 Google 계정이 없거나 scope 업그레이드가 필요하다면 **Connect Google account**를 사용하고, 그다음 **Select calendars**를 열어 동기화할 캘린더를 선택합니다.

### 시작 전에

Google Cloud Console에서 Google **Desktop app** OAuth client를 준비하세요.

직접 열기: [Google Auth platform clients](https://console.cloud.google.com/auth/clients).

필요한 값:

- Desktop app client ID
- Desktop app client secret

TUI는 Google OAuth client 설정 페이지를 열어 새 client를 만들거나 기존 값을 복사할 수 있게 합니다.

### 연결 중 일어나는 일

1. Google client ID를 붙여넣습니다.
2. Google client secret을 붙여넣습니다.
3. `syncdown`이 브라우저에서 Google 동의 화면을 엽니다.
4. 요청된 Calendar readonly scope를 승인합니다.
5. Google이 `syncdown`이 시작한 로컬 callback 서버로 리디렉션합니다.
6. `syncdown`이 승인된 scope를 검증하고 연결을 완료합니다.
7. **Select calendars**로 돌아가 동기화할 캘린더를 선택합니다.

### 공유 Google 계정과 scope 업그레이드

- Gmail과 Google Calendar는 `syncdown`에 저장된 같은 Google 계정을 사용합니다.
- 먼저 Gmail을 연결했다면 Google Calendar를 위해 추가 Calendar scope를 승인하도록 다시 연결해야 할 수 있습니다.
- 반대로 먼저 Calendar를 연결했다면 나중에 Gmail을 활성화할 때 Gmail scope를 위해 다시 연결해야 할 수 있습니다.
- 다시 연결하면 공유 Google 계정 자격 증명이 갱신되며, 커넥터별로 별도 Google 계정이 필요한 것은 아닙니다.

## 헤드리스 설정

```sh
syncdown config set googleCalendar.enabled true
syncdown config set googleCalendar.interval 1h
syncdown config set googleCalendar.selectedCalendarIds primary,work@example.com

printf '%s' "$GOOGLE_CLIENT_ID" | syncdown config set google.clientId --stdin
printf '%s' "$GOOGLE_CLIENT_SECRET" | syncdown config set google.clientSecret --stdin
printf '%s' "$GOOGLE_REFRESH_TOKEN" | syncdown config set google.refreshToken --stdin
```

## 무엇이 동기화되는가

현재 Google Calendar 커넥터는 다음을 수행합니다.

- `googleCalendar.selectedCalendarIds`에 지정된 캘린더만 동기화합니다.
- 이벤트 또는 반복 시리즈당 Markdown 파일 하나를 씁니다.
- 각 반복 인스턴스로 확장하지 않고 반복 시리즈를 시리즈 단위 문서로 유지합니다.
- 이벤트 메타데이터와 설명 본문을 저장합니다.
- 출력물을 캘린더, 연도, 월 기준으로 정리합니다.

## 증분 동작

첫 실행에서는 선택한 각 캘린더에 대해 전체 스캔을 수행하고 캘린더별 sync token을 저장합니다.

이후 실행에서는 해당 캘린더별 sync token을 사용해 변경된 이벤트만 가져옵니다.

저장된 sync token이 만료되면 `syncdown`은 해당 캘린더에 대해 전체 재구성으로 되돌아갑니다.

`googleCalendar.selectedCalendarIds`에서 캘린더를 제거하면 다음 동기화에서 그 캘린더의 로컬 Markdown 파일이 삭제됩니다.

## 설정 옵션

### `googleCalendar.selectedCalendarIds`

동기화에 포함할 캘린더 ID를 쉼표로 구분해 지정합니다.

예시:

```text
primary
primary,work@example.com
```

## 현재 제한 사항

- 선택한 캘린더만 지원
- 반복 이벤트를 occurrence 단위로 확장하지 않음
- 임의 날짜 범위 동기화 없음
- 무인 실행을 위해 유효한 Google refresh token 필요
