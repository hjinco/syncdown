# Gmail

## 인증 모델

Gmail 커넥터는 Google OAuth를 사용하며 Gmail readonly scope가 필요합니다.

```text
https://www.googleapis.com/auth/gmail.readonly
```

## 여기서 OAuth가 의미하는 것

OAuth를 사용하면 `syncdown`에 Google 계정 비밀번호를 제공하는 대신 브라우저에서 Gmail 접근을 승인할 수 있습니다.

Gmail에서는 다음을 의미합니다.

- Google **Desktop app** OAuth client를 만들거나 기존 것을 재사용합니다.
- client ID와 client secret을 `syncdown`에 입력합니다.
- `syncdown`에 필요한 Gmail scope에 대해 Google이 브라우저 동의 화면을 표시합니다.
- 이후 동기화 때마다 다시 연결하지 않아도 되도록 `syncdown`이 연결을 완료합니다.

이후 실행에서는 승인된 연결을 자동으로 계속 사용합니다.

## TUI 설정

**Connectors → Gmail**을 열고 **Connect Google account**를 선택합니다.

TUI는 다음을 요청합니다.

- Google Desktop app client ID
- Google Desktop app client secret

그 후 브라우저를 열고 로컬 callback을 기다리며, 승인된 scope를 검증한 뒤 연결을 완료합니다.

### 시작 전에

Google Cloud Console에서 Google **Desktop app** OAuth client를 준비하세요.

직접 열기: [Google Auth platform clients](https://console.cloud.google.com/auth/clients).

필요한 값:

- Desktop app client ID
- Desktop app client secret

TUI는 Google OAuth client 설정 페이지를 열어 새 client를 만들거나 기존 값을 복사할 수 있게 합니다.

### 연결 중 일어나는 일

1. Google client ID를 붙여넣습니다.
2. Google client secret을 붙여넣습니다.
3. `syncdown`이 브라우저에서 Google 동의 화면을 엽니다.
4. 요청된 Gmail readonly scope를 승인합니다.
5. Google이 `syncdown`이 시작한 로컬 callback 서버로 리디렉션합니다.
6. `syncdown`이 승인된 scope가 충분한지 검증하고 연결을 완료합니다.

### Gmail scope 관련 참고

- Gmail 동기화에는 `https://www.googleapis.com/auth/gmail.readonly`가 필요합니다.
- 나중에 Google Calendar도 활성화하면 `syncdown`은 같은 Google 계정에서 Gmail과 Calendar scope의 합집합을 요청합니다.
- 연결된 Google 계정에 새로 필요한 scope가 없다면 다시 연결해 업데이트된 동의 화면을 승인하세요.

## 헤드리스 설정

```sh
syncdown config set gmail.enabled true
syncdown config set gmail.interval 1h
syncdown config set gmail.syncFilter primary
syncdown config set gmail.fetchConcurrency 10

printf '%s' "$GOOGLE_CLIENT_ID" | syncdown config set google.clientId --stdin
printf '%s' "$GOOGLE_CLIENT_SECRET" | syncdown config set google.clientSecret --stdin
printf '%s' "$GOOGLE_REFRESH_TOKEN" | syncdown config set google.refreshToken --stdin
```

## 무엇이 동기화되는가

현재 Gmail 커넥터는 다음을 수행합니다.

- 기본적으로 `Primary` 받은편지함 메시지를 동기화합니다.
- `gmail.syncFilter=primary-important`를 사용하면 `Primary`와 `Important` 둘 다 요구할 수 있습니다.
- 메시지당 Markdown 파일 하나를 씁니다.
- 메시지 메타데이터와 본문 텍스트를 저장합니다.
- 출력 디렉터리에서 계정 이메일, 연도, 월 기준으로 메시지를 정리합니다.

## 증분 동작

첫 실행에서는 활성 Gmail 필터에서 메시지 ID를 스트리밍하며, 조건에 맞는 받은편지함 전체를 스캔합니다.

이후 실행에서는 Gmail history ID를 사용해 변경된 메시지만 가져옵니다.

저장된 Gmail history cursor가 만료되었거나 예전 필터 형식에서 온 cursor라면, `syncdown`은 전체 범위 재스캔으로 되돌아갑니다.

메시지가 사라지거나 현재 Gmail 필터에 더 이상 맞지 않으면 해당 로컬 Markdown 파일은 삭제됩니다.

## 튜닝 옵션

### `gmail.fetchConcurrency`

병렬로 가져올 메시지 수를 제어합니다.

기본값:

```text
10
```

### `gmail.syncFilter`

어떤 받은편지함 메시지가 Gmail 동기화 대상인지 제어합니다.

지원 값:

```text
primary
primary-important
```

동작:

- `primary`: `INBOX ∩ CATEGORY_PERSONAL`만 동기화
- `primary-important`: `INBOX ∩ CATEGORY_PERSONAL ∩ IMPORTANT`만 동기화

## 현재 제한 사항

- `Primary` 범위 받은편지함 동기화만 지원
- 첨부파일 내보내기 없음
- 인라인 에셋 내보내기 없음
- 무인 실행을 위해 유효한 Google refresh token 필요
