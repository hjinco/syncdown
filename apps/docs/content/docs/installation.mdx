# Installation

## Prebuilt binaries

`syncdown` publishes release binaries for:

```text
darwin-arm64
darwin-x64
linux-x64
windows-x64
```

On macOS and Linux:

```sh
curl -fsSL https://raw.githubusercontent.com/hjinco/syncdown/main/scripts/install.sh | sh
```

On Windows PowerShell:

```powershell
irm https://raw.githubusercontent.com/hjinco/syncdown/main/scripts/install.ps1 | iex
```

Manual downloads are also available from the [GitHub Releases page](https://github.com/hjinco/syncdown/releases).

## Installer environment variables

The install scripts support:

- `SYNCDOWN_VERSION` to install a specific version such as `0.1.0` or `v0.1.0`
- `SYNCDOWN_INSTALL_DIR` to override the install directory

Example:

```sh
SYNCDOWN_VERSION=v0.1.0 SYNCDOWN_INSTALL_DIR="$HOME/.local/bin" \
  curl -fsSL https://raw.githubusercontent.com/hjinco/syncdown/main/scripts/install.sh | sh
```

## Run from source

The workspace uses Bun workspaces, Turborepo, and TypeScript path aliases that resolve directly to workspace source.

Install dependencies:

```sh
bun install
```

Local development and CI target Bun `1.3.10`.

OpenTUI source builds also require Zig on the developer machine.

Common workspace tasks:

```sh
bun run build
bun run lint
bun run check
bun run typecheck
bun run test
```

Package-scoped runs can be executed through Turbo:

```sh
bun run turbo run build --filter=docs
bun run turbo run typecheck --filter=@syncdown/core
bun run turbo run test --filter=@syncdown/connector-notion
bun run turbo run test --filter=@syncdown/connector-gmail
```

## Verify the install

Once the binary is available on your `PATH`, run:

```sh
syncdown --help
syncdown status
syncdown update --check
```

If you want the full-screen setup flow, continue with [Getting Started](/docs/getting-started).

If you want a headless CLI setup flow, continue with [Configuration](/docs/configuration).

If you just want to confirm that the interactive entrypoint opens, launch:

```sh
syncdown
```
