# はじめに

このガイドは、新しい環境からフルスクリーン TUI を使って最初のローカル Markdown 同期まで進める最短ルートです。

## 1. `syncdown` をインストール

`syncdown` がまだ `PATH` にない場合は、先にプリビルド済みバイナリをインストールします。

`syncdown` は次のリリースバイナリを提供しています。

```text
darwin-arm64
darwin-x64
linux-x64
windows-x64
```

macOS と Linux:

```sh
curl -fsSL https://raw.githubusercontent.com/hjinco/syncdown/main/scripts/install.sh | sh
```

Windows PowerShell:

```powershell
irm https://raw.githubusercontent.com/hjinco/syncdown/main/scripts/install.ps1 | iex
```

手動で取得したい場合は [GitHub Releases page](https://github.com/hjinco/syncdown/releases) からダウンロードできます。

## 2. TUI を起動

対話型ターミナルを開き、次を実行します。

```sh
syncdown
```

OpenTUI アプリが起動し、次の機能が使えます。

- 出力パスとコネクタ認証のセットアップ画面
- 診断情報とシステムパス
- コネクタ実行とログ確認のための同期ダッシュボード

TTY なしで `syncdown` を実行すると、UI の代わりにヘルプが表示されます。

## 3. 出力ディレクトリを選ぶ

TUI で **Output** を開き、プリセットを選ぶかカスタムパスを入力します。

`syncdown` は保存前にパスを正規化し、そのディレクトリ配下にレンダリング済み Markdown を書き込みます。

この出力ツリー配下の Markdown ファイルとコネクタごとのフォルダは、`syncdown` が管理する出力として扱うのが無難です。生成済みの `.md` ファイルやフォルダを手で編集したり、名前変更・移動・構成の整理し直しをしたりすることは推奨されません。以後の同期やフル再同期で再生成・上書き・削除されることがあります。

## 4. ソースを接続

### Notion

**Connectors → Notion** を開き、次のいずれかのフローを選びます。

- **Connect with Token**: `syncdown` が Notion integrations ページを開き、integration token の入力を求めます。
- **Connect with OAuth**: `syncdown` が Notion public integration client ID と client secret を求め、ブラウザを開いてローカルコールバックを待ちます。

### Gmail

**Connectors → Gmail** を開き、**Connect Google account** を選択します。

TUI は Google Desktop app client ID と client secret を求め、ブラウザを開いてローカルで OAuth コールバックを完了します。

### Google Calendar

**Connectors → Google Calendar** を開きます。

共有 Google アカウントがまだ接続されていない場合は **Connect Google account** を使い、その後 **Select calendars** を開いて同期したいカレンダーを選びます。

### Apple Notes

**Connectors → Apple Notes** を開きます。

macOS では Apple Notes 同期を有効にします。初回実行時には、現在のターミナルまたはアプリに Notes へのアクセスを許可するよう macOS が求めることがあります。

## 5. 最初の同期を実行

TUI ホーム画面で **Sync** を開き、次のいずれかを選びます。

- **Run all**
- **Run Notion**
- **Run Gmail**
- **Run Google Calendar**
- **Run Apple Notes**

最近のコネクタログと実行状態は、同じダッシュボード上で確認できます。

代わりにヘッドレスな CLI ループを使いたい場合は、次を実行します。

```sh
syncdown run --watch
syncdown run --watch --interval 5m
```

`--interval` を省略した場合の既定値は `1h` です。

## 6. 結果を確認

出力ディレクトリで、コネクタごとに整理された Markdown ファイルを確認します。

CLI から現在のアプリ状態を確認することもできます。

```sh
syncdown status
syncdown connectors
syncdown doctor
```

## 次のステップ

- TUI を使わずに `syncdown` を設定したい場合は [Configuration](/docs/configuration) を参照してください。
- コマンド全体のリファレンスが必要なら [CLI](/docs/cli) を参照してください。
- コネクタごとの設定詳細が必要なら [Connectors](/docs/connectors) を参照してください。
